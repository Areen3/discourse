.policy(name = :default, &block) ⇒ Object

Evaluates a set of conditions related to the given context. If the block doesn’t return a truthy value, then the policy will fail. More than one policy can be defined and named. When that’s the case, policies are evaluated in their definition order.

Examples:

policy(:invalid_access) do
  guardian.can_delete_chat_channel?
end
